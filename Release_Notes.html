Purpose

This application is an example of a UART system bootloader for WB0x. It can be tailored for BlueNRG-2 and BlueNRG-LPS. For protocol details, please refer to AN5471. You can refer to README.md for more details. It is helpful while you are using WB0x (or similar) with a host MCU, connected through UART. In this scenario, the host can update the firmware of WB0x through UART.

The STM32L4-System-DFU-WB0 example is being delivered as-is and is not necessarily supported by ST.

Known Limitations

1. Binary file size: Full flash size flashed to BLE WB09 even the program size is less. There is no storing of binary file size.

2. Residual bytes not flashed: The frame length is 224 bytes, transferred each time and written to the WB09. The last residual binary chunk is not written to the WB09. For example, if the flash size is 512 KB = 524288 bytes, the residual bytes are calculated as 524288 % 224 = 128 bytes.
